Three approaches to the rebuild

Different trade-offs depending on how much you want to grow the site vs. keep it lean.

A

Astro — content-as-files

Projects and writing live as Markdown/MDX files in the repo. Near-zero JS, instant load, deploy to Vercel/Netlify. Update the site the same way you update code — no CMS, no database. Simple mental model forever.

Pros

  • Fastest possible site
  • No moving parts
  • Easy to maintain long-term
  • Great SEO out of the box

Cons

  • Adding content requires a code push
  • Less dynamic capability later
B

TanStack Start — full-stack React

Modern React with server functions, type-safe routing, and SSR. More complex to set up but you get a real full-stack app — contact forms, dynamic project filtering, CMS integration, or any future feature without switching stacks.

Pros

  • Full-stack power from day one
  • Current, modern stack
  • Scales to anything you want to add

Cons

  • More infrastructure to manage
  • Heavier than a portfolio needs
  • TanStack Start is still maturing
C

Astro + Islands (recommended)

Static Astro as the foundation — but with React "islands" for any interactive pieces (filterable project grid, contact form, etc). Gets you 95% of TanStack's flexibility at Astro's performance. Best of both worlds for a portfolio that might grow.

Pros

  • Fast by default, interactive where needed
  • Can use React components you already know
  • Content-as-files simplicity
  • Grows gracefully

Cons

  • Two mental models (Astro + React)
  • Not the same as a "real" React app if that matters to you
