**Date:** 2026-03-12

## Overview

Migrate the doug-march.com portfolio site from CSS Modules to PandaCSS using the `styled()` factory pattern. Simultaneously establish an `elements/` design system ŌĆö a `definePreset()` based token library structured to be shareable as a future npm package with associated sites (Spaceman, etc.).

This spec also supersedes **Tasks 1ŌĆō3 of `2026-03-12-mobile-responsive-404.md`** (not yet executed). Mobile responsive work will be implemented in PandaCSS directly rather than in CSS Modules first.

## Architecture

```
elements/
  preset.ts       ŌåÉ definePreset() ŌĆö all token definitions
  index.ts        ŌåÉ re-exports elementsPreset

app/hooks/
  useTheme.ts     ŌåÉ theme toggle hook (browser/app-specific, not in elements/)

panda.config.ts   ŌåÉ PandaCSS config: imports preset, sets up conditions + styled factory
styled-system/    ŌåÉ generated by panda codegen (gitignored)

app/styles/global.css        ŌåÉ stripped to resets only
app/routes/__root.tsx        ŌåÉ gains inline theme-init <script>
app/components/*.tsx         ŌåÉ CSS Modules deleted, styles move into styled()
app/routes/*.tsx             ŌåÉ same
```

### Why `elements/` not a `design-system/` directory

Structured from day one as a future npm package (`elements` is the intended package name). The `definePreset()` export means consuming projects add it to their own `panda.config.ts` with zero boilerplate.

### Replace, don't extend

PandaCSS ships with `@pandacss/preset-base` and `@pandacss/preset-panda`. Both are excluded. `global.css` handles resets. The `elements` preset is the only source of tokens.

## Token Architecture

Two-tier hierarchy: **primitives** (raw values) ŌåÆ **semantic tokens** (named meanings that reference primitives and switch by theme condition).

### Primitive Tokens

Scale convention: lower number = lighter value (industry standard ŌĆö `50` is lightest, `900` is darkest).

### Semantic Tokens

Reference primitives via `'{colors.ink.800}'` path syntax. Switch value by theme condition using `base` (dark, default) and `_light`.

```ts
semanticTokens: {
  colors: {
    bg: {
      DEFAULT:  { value: { base: '{colors.void.300}', _light: '{colors.ink.50}'  } },
      side:     { value: { base: '{colors.void.200}', _light: '{colors.ink.100}' } },
      card:     { value: { base: '{colors.void.100}', _light: '#FFFFFF'          } },
    },
    border: {
      DEFAULT:  { value: { base: '{colors.void.400}', _light: '{colors.ink.200}' } },
      mid:      { value: { base: '{colors.void.500}', _light: '{colors.ink.300}' } },
    },
    text: {
      DEFAULT:  { value: { base: '{colors.ink.800}',  _light: '{colors.ink.900}' } },
      mid:      { value: { base: '{colors.ink.700}',  _light: '{colors.ink.600}' } },
      dim:      { value: { base: '{colors.ink.500}',  _light: '{colors.ink.400}' } },
    },
    accent: {
      DEFAULT:  { value: { base: '{colors.cyan.600}', _light: '{colors.cyan.400}' } },
      dim:      { value: { base: '{colors.cyan.500}', _light: '{colors.cyan.500}' } },
      glow:     { value: { base: '{colors.cyan.glow}',_light: 'rgba(0,110,150,0.08)' } },
    },
    logo: {
      blue:     { value: { base: '{colors.blue.500}',  _light: '{colors.blue.400}'  } },
      blueDim:  { value: { base: '{colors.blue.dim}',  _light: 'rgba(58,127,196,0.12)' } },
      green:    { value: { base: '{colors.green.500}', _light: '{colors.green.400}' } },
      greenDim: { value: { base: '{colors.green.dim}', _light: 'rgba(74,174,58,0.12)'  } },
    },
  },
}
```

**Usage in `styled()` components:** CSS property names infer their token category, so `colors.` is dropped:

```tsx
const Card = styled('div', {
  base: {
    background: 'bg.card',
    borderColor: 'border',
    color: 'text',
  }
})
```

For compound/shorthand properties (`border`, `transition`), use longhands instead ŌĆö PandaCSS maps each longhand to the correct token category:

```tsx
// border ŌĆö use longhands
borderWidth: '1px',
borderStyle: 'solid',
borderColor: 'border',      // inferred: colors

// transition ŌĆö use longhands
transitionProperty: 'color',
transitionDuration: 'fast', // inferred: durations
transitionTimingFunction: 'default', // inferred: easings
```

Avoid shorthand properties like `border: '...'` or `transition: '...'` ŌĆö PandaCSS cannot infer token categories from shorthands and they will be emitted as literal strings, bypassing token resolution.

## Theme Switching

### Conditions in `panda.config.ts`

```ts
conditions: {
  extend: {
    light: '.light &, [data-theme=light] &',
    dark:  '.dark &,  [data-theme=dark] &',
  }
}
```

Dark is the **default** (`base` in semantic tokens). Light activates on `.light` class or `[data-theme=light]` attribute on `<html>`.

### Theme Initialization Script

An inline `<script>` injected into `<head>` in `__root.tsx` before React renders. Runs synchronously ŌĆö prevents flash of wrong theme (FOUC) on page load.

TanStack Router's `head()` function accepts a `scripts` array. Inline scripts use the `children` key:

```ts
// In __root.tsx head() return value:
scripts: [
  {
    children: `(function(){
      var s=localStorage.getItem('theme');
      var p=s||(matchMedia('(prefers-color-scheme: dark)').matches?'dark':'light');
      document.documentElement.classList.add(p);
    })();`
  }
]
```

> **Note for implementer:** Verify the `scripts` entry shape against the installed version of `@tanstack/react-router`. If `children` is not supported for inline scripts, an alternative is to render the script as a `dangerouslySetInnerHTML` element inside `RootComponent` before `<HeadContent />`.

Behavior:
- First visit, no stored preference ŌåÆ reads `prefers-color-scheme`, applies `dark` or `light` class
- Returning visit with stored preference ŌåÆ applies stored class, ignores system preference
- Toggle (future) ŌåÆ updates class + `localStorage.setItem('theme', next)`

### `useTheme()` Hook

Lives in `app/hooks/useTheme.ts` ŌĆö **not** in `elements/`. The `elements/` preset is a pure token library intended for npm distribution; it must not contain browser or React dependencies. `useTheme` directly reads `document.documentElement` and `localStorage`, making it app-specific.

```ts
// app/hooks/useTheme.ts
export function useTheme() {
  const current = (): 'dark' | 'light' =>
    document.documentElement.classList.contains('light') ? 'light' : 'dark'

  const toggle = () => {
    const next = current() === 'dark' ? 'light' : 'dark'
    document.documentElement.classList.remove('dark', 'light')
    document.documentElement.classList.add(next)
    localStorage.setItem('theme', next)
  }

  return { toggle, current }
}
```

## CSS Migration Strategy

### PandaCSS Setup

1. Install: `pnpm add -D @pandacss/dev`
2. Init: `pnpm panda init` ŌĆö generates `panda.config.ts` + `styled-system/`
3. Add `styled-system/` to `.gitignore`
4. Add `panda codegen --watch` to the Vite dev script in `package.json`

### `styled()` Pattern

PandaCSS generates a `styled` factory at `styled-system/jsx`. Components use `styled('element', config)` with `base` styles and optional `variants`.

```tsx
import { styled } from '../styled-system/jsx'

const NavLink = styled('a', {
  base: {
    fontSize: 'sm',
    fontWeight: 'bold',
    color: 'text.mid',
    transitionProperty: 'color',
    transitionDuration: 'fast',
    transitionTimingFunction: 'default',
    _hover: { color: 'accent' },
  },
  variants: {
    active: {
      true: { color: 'accent' },
    }
  }
})
```

### Responsive / Mobile

PandaCSS responsive styles use a custom `_mobile` condition. Since `@pandacss/preset-panda` is excluded, built-in responsive conditions like `smDown` are not available. A custom `mobile` condition is defined in `panda.config.ts` and used as `_mobile` in style objects:

```tsx
const SidebarRoot = styled('aside', {
  base: {
    // desktop styles
    width: '280px',
    height: '100vh',
    // mobile override ŌĆö conditions nest inside base
    _mobile: {
      width: '100%',
      height: 'auto',
    }
  }
})
```

The `mobile` condition in `panda.config.ts`:

```ts
conditions: {
  extend: {
    mobile: '@media (max-width: 767px)',
  }
}
```

The underscore prefix (`_mobile`) is the PandaCSS convention for referencing conditions as style keys.

### Migration Order

Each component is migrated atomically: delete `.module.css`, inline styles into `styled()`, verify TypeScript, commit.

Order (dependencies first):
1. Install PandaCSS + create `elements/` preset ŌĆö commit
2. `Layout` (no deps)
3. `Sidebar` (no deps) ŌĆö **includes mobile top bar responsive styles**
4. `MobileFooter` (new component ŌĆö created directly in PandaCSS, never touches CSS Modules) ŌĆö **includes iOS safe area**
5. `SectionHead`
6. `ProjectRow`
7. `MissionCard`
8. `index.tsx` (page)
9. `about.tsx` (page)
10. `work.$slug.tsx` (page)
11. `__root.tsx` not-found styles (inline into `NotFound` component)
12. Strip `global.css` to resets only

After step 12, all `.module.css` files are deleted and `global.css` contains only:
```css
*, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; }
html { font-size: 18px; }
body { font-family: 'Space Mono', monospace; -webkit-font-smoothing: antialiased; }
a { color: inherit; text-decoration: none; }
```

## `panda.config.ts` ŌĆö Full Shape

```ts
import { defineConfig } from '@pandacss/dev'
import { elementsPreset } from './elements/preset'

export default defineConfig({
  preflight: false,          // global.css handles resets
  presets: [elementsPreset], // no @pandacss/preset-base or preset-panda
  include: ['./app/**/*.{ts,tsx}'],
  exclude: [],
  outdir: 'styled-system',
  jsxFramework: 'react',

  conditions: {
    extend: {
      light:  '.light &, [data-theme=light] &',
      dark:   '.dark &,  [data-theme=dark] &',
      mobile: '@media (max-width: 767px)',
    }
  },
})
```

## Out of Scope

- Theme toggle UI (location and design deferred)
- Dark mode `prefers-color-scheme` media query CSS fallback (class-based only)
- Tablet-specific breakpoints (767px covers both)
- Extracting typography styles into PandaCSS text styles (`defineTextStyles`)
- Component recipes (`defineRecipe`) ŌĆö `styled()` factory is sufficient for this site's complexity
- Publishing `elements` to npm
