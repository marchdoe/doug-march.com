# Daily Redesign — Phase 1: Generation Pipeline

**Date:** 2026-03-12
**Status:** Spec

---

## Overview

Every day, Claude redesigns doug-march.com from scratch based on "signals" from the world — weather, sports, golf news, trending GitHub repos, news headlines. The content (projects, bio) stays fixed. The entire presentation layer is fair game.

This is inspired by CSS Zen Garden. Claude is the designer. Like a Zen Garden submitter, Claude receives a creative brief (the signals) and produces its interpretation. Unlike Zen Garden, Claude can change components, layout, everything — not just CSS.

Phase 1 builds the core generation pipeline. Signals are manually populated in a YAML file. Future phases will auto-populate signals and build the archive UI.

---

## Site Context

- TanStack Router (file-based routing)

## Signals File: `signals/today.yml`

Manually populated each day before 5am. This file is the creative brief for that day's redesign.

```yaml

  conditions: "Heavy snowstorm, 8 inches overnight, 22°F"
  feel: "brutal, isolating"
sports:
  - team: "Chicago Bulls"
    result: "Lost 89-102 to Celtics"
    notes: "Third loss in a row"
  - team: "Chicago Bears"
    result: "Off season"
    notes: "New QB signing rumored"
golf:
  - "Scottie Scheffler leads Masters by 4 strokes after round 2"
  - "Rory McIlroy struggles with putting"
github_trending:
  - repo: "vercel/ai"
    description: "AI SDK for building AI-powered applications with streaming"
    stars: 45000
    why_interesting: "New streaming UI primitives"
  - repo: "microsoft/vscode-copilot-chat"
    description: "GitHub Copilot Chat extension for VS Code"
```

The `mood_override` field is optional. When set, it overrides Claude's own interpretation of the signals' emotional register.

## Scope: What Claude Can Change

### In scope — full rewrite allowed

- `elements/preset.ts` — complete token redesign (colors, typography, spacing, animations)
- `app/components/*.tsx` — any component (Layout, Sidebar, MissionCard, ProjectRow, SectionHead, MobileFooter)
- `app/routes/index.tsx`
- `app/routes/about.tsx`
- `app/routes/work.$slug.tsx`
- `app/routes/__root.tsx`
- New files Claude wants to create in `app/components/` or `app/styles/`

### Off limits — never touch

- `app/content/` — all content data
- `app/content/types.ts` — TypeScript interfaces
- `app/routeTree.gen.ts` — auto-generated by TanStack Router
- `package.json`, `pnpm-lock.yaml` — no new packages (Claude cannot add dependencies; `@anthropic-ai/sdk` and `js-yaml` are added as devDependencies by the human author as a one-time setup step: `pnpm add -D @anthropic-ai/sdk js-yaml`)
- `panda.config.ts`, `vite.config.ts`, `postcss.config.cjs` — build infrastructure
- `tsconfig.json` — TypeScript config (Claude must fix type errors in code, not loosen compiler settings)
- `styled-system/` — generated by PandaCSS

## Generation Pipeline: `scripts/daily-redesign.js`

Node.js ESM script. Orchestrates the full pipeline in sequence:

1. Read `signals/today.yml`
2. Read site context (current file contents of all mutable files + content for reference)
3. Build Claude prompt
4. Call Claude API (`claude-opus-4-6`) using `tool_use` for structured output
5. Write generated files to disk
6. Run `pnpm build` (which runs `panda codegen` first)
7. If build fails: restore originals, add error to conversation, retry (up to 3 attempts)
8. If build passes: write archive entry, git commit, git push
9. If all 3 attempts fail: restore originals, exit with code 1 (GitHub Actions reports failure)

## Claude API Integration

Uses the Anthropic SDK with `tool_use` to get structured output. Claude must call the `submit_redesign` tool to return its work.

### Tool Definition

```javascript
{
  name: "submit_redesign",
  description: "Submit the complete site redesign",
  input_schema: {
    type: "object",
    required: ["rationale", "design_brief", "files"],
    properties: {
      rationale: {
        type: "string",
        description: "1-2 paragraphs explaining creative choices and how they connect to today's signals"
      },
      design_brief: {
        type: "string",
        description: "One-sentence design brief for the archive (e.g., 'Post-blizzard brutalism: heavy type, cold grays, claustrophobic spacing')"
      },
      files: {
        type: "array",
        description: "All files to write. Must include at minimum elements/preset.ts. Include complete file contents.",
        items: {
          type: "object",
          required: ["path", "content"],
          properties: {
            path: { type: "string", description: "Relative path from repo root" },
            content: { type: "string", description: "Complete file content" }
          }
        }
      }
    }
  }
}
```

The `files` array must contain complete file contents — not diffs or patches. The minimum required file is `elements/preset.ts`. Claude may include any number of additional mutable files.

## The Claude Prompt

### System Prompt

```
You are the daily designer for doug-march.com — a personal portfolio site for Doug March, a product designer and developer.

Every day, you redesign this site from scratch. Your brief comes from signals: what happened in the world yesterday. Weather, sports, news, what's trending on GitHub. These signals are not just data — they are your creative palette. Let them genuinely inform your design decisions.

This is inspired by CSS Zen Garden, but you have more power: you can change components, layout, typography, color, animation, density, anything in the presentation layer. The content (projects, bio) is fixed. How that content is presented is entirely yours.

Be bold. A safe redesign is a failed redesign. The person who owns this site wants to be surprised by their own website every morning.
```

### User Prompt Contents

1. Today's signals (formatted from the YAML)
2. Current content (projects list with titles, types, and years — no full descriptions needed)
3. Current file structure (list of mutable files with their current content)
4. Technical requirements: must compile with `panda codegen` + `vite build`, TypeScript must be valid, no new packages
5. An example of how a signal might translate: "if there's a blizzard, maybe the layout is dense and claustrophobic with cold desaturated blues and heavy monospace. Or maybe it's the opposite — spare, empty, quiet. You decide."

## Build Validation: `scripts/utils/build-validator.js`

```javascript
async function runBuild() {
  const result = spawnSync('pnpm', ['build'], {
    cwd: ROOT,
    encoding: 'utf8',
    timeout: 120000, // 2 minute timeout
  })

  if (result.status === 0) return { success: true }

  return {
    success: false,
    error: (result.stderr + result.stdout).slice(-3000) // last 3000 chars of error
  }
}
```

On failure, the last 3000 characters of combined stderr + stdout are captured and appended to the Claude conversation as a new user message. The script then retries by calling the API again (up to 3 total attempts). On each retry, Claude receives the full conversation history including the prior attempt's files and the build error.

---

## Retry Logic

The pipeline allows up to 3 total generation attempts. On each failed build:

1. Restore all original files from the pre-run backup
2. Append the build error to the conversation history
3. Call Claude again with the full conversation (Claude sees what it wrote and what broke)
4. Write the new files and attempt the build again

If all 3 attempts fail, restore originals and exit with code 1.

## Git Commit: `scripts/daily-redesign.js`

```javascript
async function gitCommit(date, designBrief) {
  // Configure git identity before committing (required in GitHub Actions environment)
  execSync('git config user.name "Daily Redesign"', { cwd: ROOT })
  execSync('git config user.email "redesign@doug-march.com"', { cwd: ROOT })
  execSync('git add -A', { cwd: ROOT })
  const message = `design(${date}): ${designBrief}`
  execSync(`git commit -m "${message.replace(/"/g, '\\"')}"`, { cwd: ROOT })
}
```

Commit message format: `design(2026-03-12): Post-blizzard brutalism: heavy type, cold grays`

The script handles `git add` and `git commit`. The GitHub Actions workflow handles `git push`.

## GitHub Actions Workflow: `.github/workflows/daily-redesign.yml`

```yaml
name: Daily Redesign

on:
  schedule:
    - cron: '0 10 * * *'  # 5am EST (UTC-5), 9am EDT (UTC-4)
  workflow_dispatch:
    inputs:
      dry_run:
        description: 'Dry run — generate but do not commit'
        type: boolean
        default: false

jobs:
  redesign:
    runs-on: ubuntu-latest
    timeout-minutes: 30

    steps:
      - uses: actions/checkout@v4
        with:
          token: ${{ secrets.GITHUB_TOKEN }}

      - uses: pnpm/action-setup@v4
        with:
          version: 10

      - uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: 'pnpm'

      - name: Install dependencies
        run: pnpm install

      - name: Run daily redesign
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          DRY_RUN: ${{ inputs.dry_run }}
        run: node scripts/daily-redesign.js

      - name: Push changes
        if: ${{ !inputs.dry_run }}
        run: git push origin main
```

The workflow runs on a daily schedule at 5am EST. It can also be triggered manually via `workflow_dispatch`. The `dry_run` input generates without committing — useful for testing the pipeline.

Required secrets:
- `ANTHROPIC_API_KEY` — for Claude API calls
- `GITHUB_TOKEN` — provided automatically by Actions for the push step
