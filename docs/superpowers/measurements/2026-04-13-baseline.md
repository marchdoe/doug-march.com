## Per-build cost notes (Phase 0)

The unified prompt grew to approximately 14202 characters vs 9443 pre-unification. Estimated input-token impact: +1190 tokens per production build (production path was already reading unified-designer.md). Local-dev input tokens shift to match — previously ran on the smaller embedded template, now uses the same unified content.

This cost is a one-time unification expense. Phase 1's 5% growth budget is measured against this Post-Phase-0 baseline, not against pre-Phase-0.

## Phase 1 closeout

Final prompt size: **14793 bytes** (+591 bytes from Phase-1 baseline of 14202). Under the 5% ceiling of 14912 with 119 bytes headroom remaining.

Changes shipped:
- Task 1.2: new `## Do's and Don'ts (Anti-Patterns)` section with 5 DO and 6 DO NOT bullets; ~1213 bytes of redundant prose trimmed from intro/"How to Think About This"/nav-footer-examples/content-imports to stay within budget.
- Task 1.3: prose-style accessibility section replaced with a structured version grouped into 6 categories (Contrast, Type sizes, Touch targets, Focus, Motion, Links & navigation). Strict superset of the old section's constraints; body floor tightened from 14px to 16px.

Per-build token/elapsed delta: **deferred.** Measurement requires a full LLM build. Three scheduled daily pipeline runs (next 3 days) will provide real measurements and the quality-eyeball verdict that the plan's acceptance criterion calls for. Production promotion gate holds until those land.

Open follow-ups (noted during code review, not shipped in Phase 1):
1. Audit `elements/preset.ts` to confirm `2xs`/`xs` font-size tokens meet the 12px caption floor — a designer could satisfy the token list while violating the accessibility floor if the token resolves below the floor.
2. Enforcement-hook parity: only the Contrast rule currently has "don't-dodge" escape-blocker language. Consider adding analogous hooks for Touch targets and Motion (e.g., "if a control is too small, increase padding — do not remove the tap affordance").
